# Superpowers Trigger Scenarios

These scenarios are used to validate that namespaced superpowers skills are discoverable and routable.

## Scenarios

| Prompt | Expected Skill |
| --- | --- |
| "Help me brainstorm architecture options before coding." | `superpowers:brainstorming` |
| "Run the parallel-agent dispatch workflow for this bug triage." | `superpowers:dispatching-parallel-agents` |
| "Execute this approved implementation plan in batches." | `superpowers:executing-plans` |
| "We finished the branch; run the finishing workflow." | `superpowers:finishing-a-development-branch` |
| "I got review feedback, help me process it correctly." | `superpowers:receiving-code-review` |
| "Request a code review before I continue to the next step." | `superpowers:requesting-code-review` |
| "Use subagent-driven development for this feature." | `superpowers:subagent-driven-development` |
| "Debug this systematically and avoid random fixes." | `superpowers:systematic-debugging` |
| "Use strict red/green/refactor for this change." | `superpowers:test-driven-development` |
| "Set up a clean git worktree for this effort." | `superpowers:using-git-worktrees` |
| "Explain and use the superpowers workflow first." | `superpowers:using-superpowers` |
| "Verify the fix before saying it's complete." | `superpowers:verification-before-completion` |
| "Write a detailed implementation plan for this spec." | `superpowers:writing-plans` |
| "Create a new high-quality skill following best practices." | `superpowers:writing-skills` |
