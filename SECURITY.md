# Security Policy

## Reporting A Vulnerability

Please do not open a public GitHub issue for a suspected security problem.

Instead:

1. Contact the maintainer privately through GitHub.
2. Include reproduction steps, affected files or routes, and any proof-of-concept details needed to validate the report.
3. Allow time for investigation and remediation before public disclosure.

## Scope

Security reports are especially useful for:

- auth and authorization bypasses
- secret exposure
- unsafe defaults in public deployments
- injection risks
- data access boundary failures

## What To Expect

The maintainer will acknowledge triage as quickly as possible, confirm whether the issue is in scope, and communicate remediation or disclosure timing once the problem is understood.
