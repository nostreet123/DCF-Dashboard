# Support

DCF Dashboard is a **public-preview open-source project**. Support is community-driven through GitHub; there is no paid support tier or uptime guarantee.

## Before opening an issue

1. Read [docs/ONBOARDING.md](docs/ONBOARDING.md) and try the golden path that matches your goal (mock UI, live engine, or `npm run demo:compute`).
2. Check [docs/verification.md](docs/verification.md) for the latest known-good harness evidence on `main`.
3. Search [existing issues](https://github.com/nostreet123/DCF-Dashboard/issues) for duplicates.

## Where to ask for help

| Need | Channel |
|------|---------|
| Bugs, features, docs gaps | [GitHub Issues](https://github.com/nostreet123/DCF-Dashboard/issues) — use the issue templates |
| Security vulnerabilities | [SECURITY.md](SECURITY.md) — **do not** file public issues for secrets or exploits |
| Contribution workflow | [CONTRIBUTING.md](CONTRIBUTING.md) |
| Release / version questions | [CHANGELOG.md](CHANGELOG.md), [RELEASING.md](RELEASING.md) |

## What to include in a bug report

- OS and Node/Python versions (`node -v`, `python --version`)
- Whether you used mock demo mode or the Python engine
- Commands run and **redacted** logs (no API keys, `.env` contents, or internal URLs)
- Expected vs actual behavior

## Response expectations

- Maintainers triage when available; there is **no SLA** for issue or PR response time.
- Pull requests from contributors are welcome; see [CONTRIBUTING.md](CONTRIBUTING.md) for the verification matrix.
- For investment or valuation methodology questions, treat outputs as educational only — see the disclaimer in [README.md](README.md).

## Out of scope for this repository

- Hosted SaaS operations, billing, or org administration
- Personalized financial advice
- Guaranteed compatibility with private forks of Convex, Vercel, or third-party data contracts
